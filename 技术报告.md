### 1.2 Agent 工作流详解

```
用户输入: "3号线贴片机报警E302，吸嘴连续抛料"
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 1: InputParser Agent                                       │
│ 输出: {                                                         │
│   "device_model": "SMTX100",                                   │
│   "fault_code": "E302",                                        │
│   "fault_phenomenon": "抛料、报警",                             │
│   "production_line": "3号线",                                   │
│   "is_stopped": true                                            │
│ }                                                               │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 2: KnowledgeRetriever Agent                                │
│ 检索策略:                                                        │
│ 1. 按 fault_code="E302" 过滤                                    │
│ 2. 向量相似度检索 "真空压力不足"                                 │
│ 3. 返回 Top-5 相关文档片段                                       │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 3: Diagnosis Agent (LLM推理)                               │
│ 基于检索到的知识 + LLM推理:                                       │
│ • 故障名称: 真空压力不足                                          │
│ • 可能原因: 吸嘴堵塞(高)、过滤器污染(高)、气路漏气(中)...         │
│ • 推荐步骤: 检查吸嘴→检查压力表→检查过滤器...                    │
│ • 风险等级: 中                                                   │
│ • 建议备件: 吸嘴 NZ-14A, 过滤芯 VF-02                            │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 4: Escalation Agent (判断是否需要人工介入)                  │
│ 判断条件:                                                        │
│ • 停机时间 > 15分钟? 否                                          │
```

### 2.2 核心技术使用方式

#### 2.2.1 LangGraph Multi-Agent 架构

```python
# agents/graph.py
from langgraph.graph import StateGraph, END

# 定义状态
class DiagnosisState(TypedDict):
    user_input: str
    fault_context: Dict
    retrieved_docs: List
    diagnosis: Dict
    evidence_chain: List
    escalation: Dict
    final_report: Dict

# 构建工作流
workflow = StateGraph(DiagnosisState)

# 添加节点
workflow.add_node("input_parser", input_parser_node)
workflow.add_node("knowledge_retriever", knowledge_retriever_node)
workflow.add_node("diagnosis", diagnosis_node)
workflow.add_node("escalation", escalation_node)
workflow.add_node("evidence", evidence_node)
workflow.add_node("sink", sink_node)

# 定义边
workflow.set_entry_point("input_parser")
workflow.add_edge("input_parser", "knowledge_retriever")
workflow.add_edge("knowledge_retriever", "diagnosis")
workflow.add_conditional_edges(
    "diagnosis",
    should_escalate,
    {True: "escalation", False: "evidence"}
)
workflow.add_edge("escalation", "evidence")
workflow.add_edge("evidence", "sink")
workflow.add_edge("sink", END)

# 编译
app = workflow.compile()
```

#### 2.2.2 ChromaDB 向量检索

```python
# knowledge_base/chroma_store.py
import chromadb
from chromadb.config import Settings

class ChromaStore:
    def __init__(self):
        self.client = chromadb.PersistentClient(
            path="./chroma_db",
            settings=Settings(anonymized_telemetry=False)
        )
        self.collection = self.client.get_or_create_collection(
            name="intellifix_kb",
            embedding_function=DashScopeEmbeddingFunction()  # 阿里百炼 Embedding
        )
    
    def query(self, query_text: str, n_results: int = 5):
        # 使用阿里百炼 API 生成查询向量
        query_embedding = get_embeddings_from_dashscope([query_text])[0]
        return self.collection.query(
            query_embeddings=[query_embedding],
            n_results=n_results
        )
```

#### 2.2.3 阿里百炼 API 集成

```python
# config.py
class Settings(BaseSettings):
    dashscope_api_key: str = Field(alias="DASHSCOPE_API_KEY")
    dashscope_base_url: str = "https://dashscope.aliyuncs.com/compatible-mode/v1"
    dashscope_embedding_model: str = "text-embedding-v3"
    dashscope_llm_model: str = "qwen-plus"

# llm_client.py
from openai import OpenAI

def get_llm_client():
    return OpenAI(
        api_key=settings.dashscope_api_key,
        base_url=settings.dashscope_base_url
    )

def get_embeddings(texts: list) -> list:
    client = get_llm_client()
    response = client.embeddings.create(
        model=settings.dashscope_embedding_model,
        input=texts
    )
    return [item.embedding for item in response.data]
```

## 4. 产品创新点

### 4.1 技术创新

#### 4.1.1 Multi-Agent 协作诊断架构

区别于单一大模型对话，智修Agent 采用 **6-Agent 协作架构**：

- **专业化分工**: 每个 Agent 负责特定任务（解析、检索、诊断、判断、证据、输出）
- **可观测性**: 每个步骤的中间状态可追溯、可审计
- **可扩展性**: 新 Agent 可无缝接入工作流

#### 4.1.2 混合检索策略

```
检索精度 = 向量语义相似度 + Metadata 过滤 + 故障代码匹配
```

- 向量检索: 理解自然语言描述的故障现象
- Metadata 过滤: 按设备型号、故障代码精确筛选
- 混合策略: 兼顾召回率和准确率

#### 4.1.3 证据链自动生成

每个诊断建议自动标注知识来源：
- 引用具体文档名称和段落
- 支持点击跳转到原始文档
- 满足制造业审计合规要求

### 4.2 产品创新

#### 4.2.1 故障上下文自动提取

用户只需用自然语言描述故障，系统自动提取：
- 设备型号 (Device Model)
- 故障代码 (Fault Code)
- 故障现象 (Phenomenon)
- 生产线位置 (Line)
- 停机状态 (Status)

#### 4.2.2 智能升级判断

系统自动判断是否需要人工升级：
- 停机时间阈值监控
- 风险等级评估
- 所需资质判断
- 自动推荐升级路径

#### 4.2.3 结构化诊断报告

输出符合工业标准的诊断报告：
```json
{
  "fault_name": "真空压力不足",
  "probable_causes": [
    {"rank": 1, "cause": "吸嘴堵塞", "confidence": "高", "basis": [...]}
  ],
  "recommended_steps": [...],
  "spare_parts": [{"name": "吸嘴", "model": "NZ-14A"}],
  "risk_level": "中",
  "estimated_repair_time": "30分钟"
}
```

### 4.3 工程创新

#### 4.3.1 纯 API 架构，零本地模型依赖

- 所有 AI 能力通过阿里百炼 API 提供
- 无需本地 GPU 或模型部署
- 降低硬件成本，简化运维

#### 4.3.2 前后端分离，独立部署

- 前端: 静态资源，可部署到 CDN
- 后端: FastAPI 服务，支持容器化
- 知识库: ChromaDB 持久化存储
